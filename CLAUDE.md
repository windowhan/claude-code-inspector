# Claude Code Inspector — Project Guidelines

## Architecture

Single Rust binary (`claude-code-hook`) running two tokio listeners:
- **Port 7878** — HTTP proxy that intercepts all Claude Code → Anthropic API traffic
- **Port 7879** — Dashboard web server (API + built frontend assets)
- **`mcp` subcommand** — stdio MCP server so Claude Code can query inspector data

Frontend lives in `frontend/` (Vite + vanilla JS). Build output goes to `src/assets/dist/` and is embedded into the binary at compile time via `include_dir!`.

```
frontend/          Vite project (npm run build → src/assets/dist/)
src/
  main.rs          CLI (clap), server startup, dual listener
  proxy.rs         Core: intercept, forward, SSE tee, DB persist
  sse_tee.rs       Async SSE stream tee + parser
  db.rs            SQLite schema, insert/update/query
  session.rs       macOS lsof: TCP port → PID → CWD resolution
  mcp.rs           MCP server (stdio JSON-RPC, tools: list_sessions/list_requests/get_request)
  dashboard.rs     HTTP routes: /api/*, /events, static asset serving
  types.rs         Shared types: RequestRecord, SessionRecord, AppState
  assets/
    dashboard.html  Fallback single-file dashboard (always compiled in)
    dist/           Built frontend (gitignored, generated by npm run build)
```

## Setup

```bash
# Build backend
cargo build --release

# Build frontend
cd frontend && npm install && npm run build && cd ..

# Run
./target/release/claude-code-hook

# Set env for Claude Code
export ANTHROPIC_BASE_URL=http://127.0.0.1:7878

# Register MCP server with Claude Code
claude mcp add claude-inspector -- /path/to/claude-code-hook mcp
```

## Test Coverage Policy

**Maintain 100% test coverage at all times.**

- Every public function and module must have corresponding unit or integration tests.
- Tests live in the same file under `#[cfg(test)]` modules, or in `tests/` for integration tests.
- Run tests before committing: `cargo test`
- Use `cargo llvm-cov` or `cargo tarpaulin` to measure coverage; coverage must not drop below 100% on non-trivial logic.
- Mocking external I/O (SQLite, lsof, upstream HTTP) is acceptable when it enables deterministic tests.
- Do not commit code that decreases coverage. If a PR removes tests without removing the corresponding code, it should be rejected.

```bash
# Check coverage (requires cargo-llvm-cov)
cargo llvm-cov --all-features --workspace --lcov --output-path lcov.info
cargo llvm-cov report
```

## Code Style

- All warnings must be resolved before committing (`cargo build` must produce zero warnings).
- Use `tracing::{info, warn, debug, error}` for logging; no `println!` in library code.
- Sensitive headers (`x-api-key`) must never be written to the database.
- Keep `proxy.rs` and `sse_tee.rs` free of dashboard/UI concerns.
- Keep `dashboard.rs` free of proxy/forwarding logic.
- Frontend JS is plain ES modules (no bundled framework); keep it readable without a build step.

## Frontend Development

```bash
cd frontend
npm install
npm run dev    # Vite dev server on :5173, proxies /api and /events to :7879
npm run build  # Builds to ../src/assets/dist/ for embedding in binary
```

## MCP Tools Available to Claude Code

After `claude mcp add claude-inspector -- claude-code-hook mcp`:

| Tool | Description |
|------|-------------|
| `list_sessions` | All sessions with request counts + token totals |
| `list_requests` | Recent requests (filter by session_id, paginate) |
| `get_request`   | Full detail: messages, response, usage, timing |
